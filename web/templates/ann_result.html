인공신경망(scikit-learn) 상세 결과

결과 {{result}} 알고리즘이 판단하였을 때에 해당 계좌가 사기 계좌인지 아닌지를 나타내는 결과
확률 {{rate}} 알고리즘이 판단하였을 때에 결과에 따라 해당 계좌가 사기 계좌나 정상 계좌일 확률
알고리즘 정확도 {{score}} 해당 알고리즘이 해당 계좌가 사기계좌인지 아닌지를 얼마나 정확하게 탐지하는지 나타내는 정확도
최적의 파라미터 {{param}} 알고리즘의 정확도를 가장 최적으로 만들어주는 설정값
Confusion Matrix
  • True Positive(TP) : 실제 True인 정답을 True라고 예측 (정답)
  • False Positive(FP) : 실제 False인 정답을 True라고 예측 (오답)
  • False Negative(FN) : 실제 True인 정답을 False라고 예측 (오답)
  • True Negative(TN) : 실제 False인 정답을 False라고 예측 (정답)
정밀도(Precision) {{precision}} 알고리즘이 사기계좌로 예측한 것들 중에 실제로 사기계좌인 비율
재현율(Recall) {{recall}} 실제 사기계좌인 것들 중에 알고리즘이 사기계좌라고 예측한 비율
F1-score {{f}} 정밀도(precision)와 재현율(recall)의 조화평균. 정밀도가 높다고 무조건 좋은 알고리즘도 아니고 재현율이 높다고 무조건 좋은 모형이 아니다. 정밀도만 높고 재현율이 낮다면... 그래서 그 둘의 조화평균인 F1-score를 보아 좋은 알고리즘인지 아닌지를 판단한다. F1-score가 높을 수록 좋은 알고리즘이다.
True-Positive Rate(TPR) {{tpr}} 실제 사기계좌인 것들 중에 알고리즘이 사기계좌라고 제대로 예측한 비율(재현율과 동일함)
False-Positive Rate(FPR) {{fpr}} 실제 사기계좌가 아닌 것들 중에 알고리즘이 사기계좌라고 잘못 예측한 비율.
Receiver Operating Characteristic(ROC) Curve 여러 임계값들을 기준으로 TPR과 FPR의 변화를 그린 그래프. 커브가 왼쪽 윗모서리에 가까울 수록 더 좋은 알고리즘이다.
Area Under Curve(AUC) {{auc}} ROC Curve 아래의 면적값. 최대값은 1이며 1에 가까울 수록 더 좋은 알고리즘이다. 면적이 1에 가깝다는 것은 FPR에 비해 TPR 값이 크다는 것, 즉, 잘못 예측한 비율보다 제대로 예측한 비율이 더 높다는 뜻이다.
